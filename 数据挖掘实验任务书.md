# 数据挖掘实验任务书：旅游攻略知识图谱与智能助手

**实验背景**：本实验系列旨在构建一个面向旅游领域的智能问答与规划系统。
**数据集来源**：本实验采用包含 **泰山、西湖、张家界** 三个著名景区的游记与攻略数据（模拟自 IndustryCorpus2_tourism_geography 等开源数据集的结构），区别于原实验指导书中的“故宫、黄山、九寨沟”数据集，以验证算法的泛化能力。

---

## 实验项目1：景区指南数据采集与预处理

- **大致时间**：45分钟
- **实验目标**：完成数据的清洗与结构化，为后续任务提供标准输入。
- **主要步骤与过程描述**：
    1. **数据准备**：
        - 选取泰山、西湖、张家界三个景区，分别收集其官方推荐路线（如“泰山红门徒步线”）及5篇对应的游客游记。
        - 将原始数据整理至 `task1_data_collection/data/data.xlsx` 中，包含 `景区名称`、`官方游览路线`、`游客游记1-5` 等字段。
    2. **编写清洗脚本**：
        - 使用 `preprocess.py` 脚本读取 Excel 文件。
        - 应用正则表达式去除无关字符（如 HTML 标签、特殊符号），统一标点符号，处理空白字符。
        - 将非结构化的文本转换为结构化的 JSON/Excel 格式。
    3. **执行与验证**：
        - 运行命令：`python preprocess.py`
        - 检查输出文件 `task1_data_collection/data/data_cleaned.xlsx`，确认数据整洁且格式正确。

> **[此处插入截图：data_cleaned.xlsx 的内容预览，展示清洗后的数据结构]**

- **差异说明**：
    - 原计划使用爬虫实时采集，考虑到网络稳定性与时间限制，本实验直接使用了预置的模拟数据集，集中精力于数据清洗逻辑。

## 实验项目2：游览步骤的实体识别

- **大致时间**：60分钟
- **实验目标**：从文本中提取 **景点(POI)**、**交通方式**、**时间** 三类关键实体。
- **主要步骤与过程描述**：
    1. **构建领域词典**：
        - 在 `task2_entity_recognition/custom_dicts/` 下创建 `poi/` 目录。
        - 分别整理 `taishan.txt`、`xihu.txt`、`zhangjiajie.txt`，录入“南天门”、“断桥”、“金鞭溪”等专有景点名词。
        - 更新 `transport.txt`（索道、大巴）和 `time.txt`（上午、2小时）。
    2. **实体提取**：
        - 运行 `python entity_extraction.py`。
        - 脚本加载清洗后的数据，利用 Jieba 分词加载自定义词典，对文本进行分词与词性标注。
        - 提取匹配到的实体，生成 `output/entity_results.json`。
    3. **可视化分析**：
        - 运行 `python generate_wordcloud.py`。
        - 生成各景区的实体词云图，直观展示高频景点与交通方式。

## 实验项目3：游览路线的层级结构挖掘

- **大致时间**：60分钟
- **实验目标**：解析路线文本，构建“时间-空间-活动”的层级结构，并对比官方与游客路线的差异。
- **主要步骤与过程描述**：
    1. **路线解析**：
        - 编写 `route_parser.py`，利用正则表达式识别“上午”、“下午”、“然后”、“接着”等时间与逻辑连接词。
        - 将线性文本解析为树状结构（如：Day 1 -> 上午 -> 景点A -> 活动B）。
    2. **层级构建**：
        - 运行 `python main_task3.py`。
        - 脚本自动处理所有景区的官方路线与游记，生成层级结构图。
    3. **对比分析**：
        - 脚本计算官方路线与游客实际路线在景点重合度、顺序一致性上的差异。
        - 生成对比报告 `comparisons/comparison_report.json`。

## 实验项目4：程序性知识的共指消解

- **大致时间**：45分钟
- **实验目标**：解决游记中“它”、“那里”等代词的指代消解问题，提高信息抽取的准确性。
- **主要步骤与过程描述**：
    1. **代词检测**：
        - 使用 `coref_extractor.py` 中的正则规则，定位文本中的代词及其上下文。
    2. **消解策略**：
        - 采用基于规则的“最近前修饰名词”策略（Nearest Antecedent Strategy）。
        - 结合实验2识别的实体列表，优先将代词链接到最近的 POI 实体。
    3. **执行与评估**：
        - 运行 `python main.py`。
        - 脚本输出消解前后的文本对比，并自动计算消解覆盖率。

## 实验项目5：条件性游览建议的抽取

- **大致时间**：60分钟
- **实验目标**：挖掘“如果...则...”形式的条件建议，分析不同游客群体（老人、亲子等）的需求差异。
- **主要步骤与过程描述**：
    1. **模式定义**：
        - 在 `config/condition_patterns.json` 中定义正则模式，如 `如果(?P<condition>.*?)，建议(?P<advice>.*?)`。
        - 在 `config/visitor_type_patterns.json` 中定义游客类型关键词（如“带孩子”->“亲子”）。
    2. **抽取与分类**：
        - 运行 `python main.py`。
        - 脚本遍历所有文本，抽取条件建议对，并根据关键词将建议归类（交通建议、装备建议等）。
    3. **可视化展示**：
        - 生成不同游客类型的建议分布图，分析特定人群的关注点。

## 实验项目6：多源程序性知识的融合与图谱可视化

- **大致时间**：60分钟
- **实验目标**：将前述实验提取的实体、路线、建议融合为统一的知识图谱。
- **主要步骤与过程描述**：
    1. **数据融合**：
        - 编写 `knowledge_fusion.py`，读取 Task 2 的实体、Task 3 的路线结构、Task 5 的建议。
        - 以“景点”为核心节点，将“交通方式”、“游玩时间”、“相关建议”作为属性或关联节点进行链接。
    2. **图谱构建**：
        - 使用 `NetworkX` 库构建图结构。
        - 定义节点类型（Spot, Activity, Time）和边类型（Next, Has_Advice）。
    3. **可视化**：
        - 运行 `python main.py`。
        - 生成各景区的知识图谱可视化图片，展示景点间的流转关系与相关知识。

## 实验项目7：程序性知识应用：旅游规划 Agent

- **大致时间**：90分钟
- **实验目标**：构建一个能够理解自然语言并利用知识图谱进行推理的智能助手。
- **主要步骤与过程描述**：
    1. **系统架构搭建**：
        - 采用 **Go 语言** 结合 **CloudWeGo Eino** 框架构建后端服务。
        - 集成 **智谱 AI (ZhipuAI)** 的大模型 API 作为推理核心。
    2. **工具封装**：
        - 编写 `knowledge_graph_tool.go`，将 Task 6 生成的 JSON 图谱封装为可调用的查询工具（Tool）。
        - 定义工具的输入输出 Schema，使 LLM 能够准确调用。
    3. **Agent 编排**：
        - 在 `main.go` 中编排 ReAct（Reasoning + Acting）流程。
        - 实现“用户提问 -> LLM 思考 -> 调用图谱工具 -> 获取信息 -> LLM 生成回答”的闭环。
    4. **交互实现**：
        - 启动 Web 服务（端口 8081），在浏览器中进行对话测试。
        - 测试案例：“带老人去泰山怎么玩？”、“西湖有哪些适合拍照的地方？”。

- **差异说明**：
    - 原实验指导书建议使用 Python/LangChain 框架。本实验创新性地使用了 **Go 语言** 生态（CloudWeGo Eino），验证了多语言环境下构建 Agent 的可行性，并实现了更高性能的服务端响应。
