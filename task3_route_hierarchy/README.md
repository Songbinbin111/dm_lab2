# 实验项目3：游览路线的层级结构挖掘

## 对应科研点
面向程序性文本的层级结构挖掘技术。

## 实验目标
挖掘出游览路线的层次结构，理解官方指南与游客游记中路线组织的差异。

## 实验主要内容
1. **官方路线结构化解析**：分析官方推荐路线的文本描述方式，归纳其格式规律（如按时间顺序、按景点分段），设计解析规则，将每条路线转换为统一的结构化表示（如包含步骤序号、时间、景点、活动等）。
2. **时间维度层级构建**：基于解析结果，构建以时间（上午、中午、下午）为维度的游览层级结构，输出结构化的路线数据。
3. **高阶任务**：选取同一景区的官方推荐路线和游客游记中隐含的实际路线，对比两者的层级结构差异（如时间分配、景点顺序、停留时长等），并分析原因。

## 实验技术方法
1. **文本解析**：正则表达式、字符串模式匹配。
2. **数据结构**：Python字典、列表，可输出JSON或CSV。
3. **对比分析**：手动或编写脚本计算结构相似度指标。

## 实验主要步骤
1. 选取1-2个景区的官方路线文本，分析其描述模式，编写解析函数，输出结构化JSON。
2. 基于解析结果，构建时间维度层级（上午/中午/下午），并整合为完整路线。
3. 若对比官方与游客路线结构，记录差异点。
4. 整理输出文件（结构化数据、对比分析报告）。

---

## 脚本操作指南

### 文件结构
- `main_task3.py`: 主分析程序
- `route_parser.py`: 路线解析模块
- `route_analyzer.py`: 对比分析模块

### 运行步骤

1. **运行完整分析**：
   ```bash
   python main_task3.py
   ```

2. **结果检查**：
   - **层级结构**：查看 `hierarchies/` 目录下的 JSON 数据和 PNG 图片（包含张家界、泰山、西湖的路线层级结构）。
   - **对比报告**：查看 `comparisons/` 目录下的报告（JSON 数据和 PNG 图表）。
