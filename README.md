# 旅游攻略知识图谱与智能助手项目教学文档

本教程将引导您完成从源码获取、数据处理到构建一个能够智能规划路线的旅游 Agent 的全过程。整个项目分为 8 个核心阶段（预备任务 + 实验项目 1-7），每个任务都建立在前一个任务的基础之上。

---

## 预备任务：环境准备与源码获取
**目标**：配置本地开发环境并获取项目源代码。
**执行位置**：您希望存放项目的本地根目录（如 `E:\dm_lab`）。

- **第 1 步：克隆 GitHub 仓库**
  打开终端，克隆项目到本地：
  ```bash
  git clone https://github.com/Songbinbin111/dm_lab.git
  cd dm_lab
  ```
- **第 2 步：安装 Python 环境**
  建议使用 Python 3.8+。建议创建虚拟环境以保持环境整洁。
- **注意事项**：
  - 确保已安装 `git` 工具。
  - Windows 用户建议在 PowerShell 或 Git Bash 中操作。
  - 若克隆速度慢，可考虑使用国内镜像或直接下载 ZIP 包。
- **预计耗时**：10 - 15 分钟

## 实验项目1：景区指南数据采集与预处理（1学时）
- **对应科研点**：基础数据工程（支撑所有后续研究）。
- **实验目标**：获取景区指南相关数据，并处理成合适的格式，为后续知识抽取与挖掘奠定数据基础。
- **实验主要内容**：
    1. **数据采集**：使用自动化工具采集3个景区的官方游览推荐路线和对应景区的游客游记各5篇。
    2. **数据预处理**：对采集的文本进行清洗，包括去除HTML标签、元数据、特殊字符，统一空白格式，并将数据整理为结构化文件。
- **实验技术方法**：
    1. **爬虫工具**：browser-use、requests+BeautifulSoup、Selenium等。
    2. **数据清洗**：正则表达式、Python字符串处理、pandas库。
    3. **格式统一**：Excel或CSV文件存储。
- **实验主要步骤**：
    1. 选择数据获取方式（爬虫或使用提供的数据集）。
    2. 编写Python脚本对原始文本进行清洗与格式化。
    3. 检查数据质量并保存最终数据集。
- **详细文档**：请参考 [task1_data_collection/README.md](task1_data_collection/README.md)

---

## 实验项目2：游览步骤的实体识别（1学时）
- **对应科研点**：基于大语言模型和归纳推理的实体类型识别。
- **实验目标**：识别程序性文本中的核心要素（景点POI、交通方式、时间节点）。
- **实验主要内容**：
    1. **实体识别**：使用Jieba分词+自定义词典识别三类实体。
    2. **归纳推理**：对比多个景区的识别结果，归纳出适用于景区领域的实体类型体系。
    3. **可视化分析**：绘制各类实体的词频统计图（词云）。
    4. **成果输出**：输出实体定义规范表、JSON格式的实体文件、词频可视化图表。
- **实验技术方法**：
    1. **分词工具**：Jieba分词。
    2. **可视化**：Matplotlib、WordCloud库。
    3. **数据存储**：JSON格式。
- **实验主要步骤**：
    1. 构建Jieba自定义词典，实现基于词典的实体识别。
    2. 统计实体词频，绘制词云。
    3. 整理实体定义规范表，导出JSON文件。
- **详细文档**：请参考 [task2_entity_recognition/README.md](task2_entity_recognition/README.md)

## 实验项目3：游览路线的层级结构挖掘（1学时）
- **对应科研点**：面向程序性文本的层级结构挖掘技术。
- **实验目标**：挖掘出游览路线的层次结构，理解官方指南与游客游记中路线组织的差异。
- **实验主要内容**：
    1. **官方路线结构化解析**：分析官方推荐路线的文本描述方式，归纳其格式规律。
    2. **时间维度层级构建**：构建以时间（上午、中午、下午）为维度的游览层级结构。
    3. **高阶任务**：对比官方推荐路线和游客游记中隐含的实际路线的层级结构差异。
- **实验技术方法**：
    1. **文本解析**：正则表达式、字符串模式匹配。
    2. **数据结构**：Python字典、列表，可输出JSON或CSV。
    3. **对比分析**：手动或编写脚本计算结构相似度指标。
- **实验主要步骤**：
    1. 编写解析函数，输出结构化JSON。
    2. 构建时间维度层级。
    3. 对比官方与游客路线结构。
- **详细文档**：请参考 [task3_route_hierarchy/README.md](task3_route_hierarchy/README.md)

## 实验项目4：程序性知识的共指消解（1学时）
- **对应科研点**：先行词共指消解技术。
- **实验目标**：解决文本中代词（它、这、那里）指代不明的问题。
- **实验主要内容**：
    1. **代词检测**：检测出所有代词及其所在句子。
    2. **指代消解实现**：采用基于最近名词匹配的规则或模型进行消解。
    3. **效果评估**：对消解结果进行人工验证，统计准确率。
- **实验技术方法**：
    1. **代词检测**：正则表达式、Jieba分词。
    2. **指代消解**：规则或模型。
    3. **评估**：人工抽样检查。
- **实验主要步骤**：
    1. 编写代词检测脚本。
    2. 实现并运行指代消解方法。
    3. 人工评估准确性并总结。
- **详细文档**：请参考 [task4_coref_resolution/README.md](task4_coref_resolution/README.md)

## 实验项目5：条件性游览建议的抽取（1学时）
- **对应科研点**：程序性知识的复杂依赖关系抽取。
- **实验目标**：挖掘文本中的条件逻辑，如“如果下雨，则...”。
- **实验主要内容**：
    1. **条件触发词识别**：识别表示条件的触发词，提取条件句中的条件部分和建议部分。
    2. **映射表构建**：将提取的“条件→建议”对整理成结构化映射表。
    3. **高阶分析**：分析不同类型游客获得的建议差异，并可视化展示。
- **实验技术方法**：
    1. **规则抽取**：正则表达式匹配条件句型。
    2. **数据分析与可视化**：Pandas、Matplotlib。
- **实验主要步骤**：
    1. 编写规则抽取条件性建议。
    2. 人工校验并完善映射表。
    3. 对映射表按游客类型分组统计，绘制对比图表。
- **详细文档**：请参考 [task5_conditional_advice/README.md](task5_conditional_advice/README.md)

## 实验项目6：多源程序性知识的融合与图谱可视化（1学时）
- **对应科研点**：知识图谱构建与表示。
- **实验目标**：将不同来源（官网、游记）的知识整合，并进行图形化展示。
- **实验主要内容**：
    1. **数据融合**：将实验3的路线数据与实验2的实体数据融合，建立关联。
    2. **图谱构建**：构建知识图谱，定义节点类型和边类型。
    3. **可视化**：绘制知识图谱，展示游览知识网络。
- **实验技术方法**：
    1. **图数据库**：Neo4j 或 NetworkX。
    2. **可视化**：Neo4j Browser、Matplotlib。
    3. **数据整合**：Python脚本。
- **实验主要步骤**：
    1. 整合数据，生成节点和边的列表。
    2. 构建图谱并生成可视化。
    3. 分析图谱中的核心节点和路径。
- **详细文档**：请参考 [task6_knowledge_fusion/README.md](task6_knowledge_fusion/README.md)

## 实验项目7：程序性知识应用：旅游规划Agent（2学时）
- **对应科研点**：基于知识图谱和LLM的问答与生成。
- **实验目标**：设计并实现一个“旅游规划Agent”，理解用户请求并生成个性化游览路线。
- **实验主要内容**：
    1. **Agent框架设计**：采用ReAct模式，具备循环推理与工具调用能力。
    2. **功能实现**：意图解析、图谱查询、路线规划、LLM集成、交互界面。
- **技术方法**：
    1. **LLM调用**：智谱AI API。
    2. **开发框架**：Go 语言 + CloudWeGo Eino。
    3. **工具封装**：知识图谱查询工具。
- **实验主要步骤**：
    1. 搭建 Agent 框架。
    2. 实现意图解析与图谱查询。
    3. 测试并优化 Agent 决策逻辑。
- **详细文档**：请参考 [task7_agent/README.md](task7_agent/README.md)
